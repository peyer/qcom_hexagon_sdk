cdep User's Guide

Command-line Usage 

Usage:  cdep [option | filename]...
       
Filenames specify source files to scan for dependencies.

Options:
    -M         : Output dependency lines in 'make' format.
    -M+        : Like -M, but includes an additional empty dependency
                 line for each included file.  This allows 'make' to
                 proceed when the included file does not exist (which
                 can occur when dependency files are stale).
    -1         : Output simple list of included files.
    -S<d><f>   : Output statistics.  <d> and <f> are optional.
                 Characters in <f> denote fields:
                    'F' = source file name
                    'B' = source file size in bytes
                    'L' = source file size in lines
                    'S' = source file score (lines not blank or comments)
                    'n' = number of included files
                    'b' = total bytes in included files
                    'l' = total lines in included files
                    's' = total score for included files
                 Default for <f> is 'FBLnbls'.  <d> is a non-alphanumeric
                 separator character; default is space.

    -I<dir>    : Add <dir> to include search path.
    -I <dir>   : Same as -I<dir>.
    -o <obj>   : Specify name for object file (LHS of dependency line).
    -o <dir/>  : A trailing slash specifies a directory for objects.
                 Object names are contructed from source names by
                 replacing the extension with '.o'.
    -f <file>  : Read file names or -I<dir> directives from <file>.
    -i <incs>  : Read list of include directories from <incs>.
    -w         : Show warnings.
    -n         : Do not recurse into included files.
    -s         : Strip path from source/object file name in output.
    -x...      : Diagnostic switches (see source code).

Default output mode = -M+
