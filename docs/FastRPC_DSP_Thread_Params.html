FastRPC DSP thread parameters

remote_session_control
Restrictions

Overview

The underlying principle of FastRPC is that every CPU process/thread making an RPC call will have a corresponding user process/thread on the DSP. Until recently, the priority and stack size of the user threads created by FastRPC on the DSP were hardcoded to 192 and 16 KB respectively. However, from Hana onward, we have introduced a new API that allows clients to configure the stack size and priority of all threads in their user process on the DSP.

remote_session_control

The following API and data structure (exposed in the “remote.h” header) can be used by clients to configure their DSP thread parameters: 

struct remote_rpc_thread_params {
        int domain; 
        int prio; 
        int stack_size;
};

Restrictions

  • This API should be called before making any other RPC call from your CPU process i.e. this should be the first call from your app.

  • Thread stack size should be between 16 KB and 8 MB.

  • DSP thread priority should be between 1 and 254 (with 1 being the highest).

    • We do not recommend using priority 255, as you may get pre-empted by interrupts even if other threads are idle. 255 is basically

      treated as the idle priority.

  • Currently, we do not support individual thread configuration i.e. if you update the thread priority and stack size once, the same parameters will be used for ALL the fastrpc threads in that DSP user process.

  • After the first RPC call has been made, the fastrpc thread parameters cannot be updated again and will persist for the lifetime of the DSP process.

    • You can always spawn “worker” threads anytime from the DSP with a different stack size and priority.

Examples

Set priority to 100 and stack size to 4 MB on CDSP 

#include "remote.h"
...
main() {
        struct remote_rpc_thread_params th_data;
        th_data.domain = CDSP_DOMAIN_ID;
        th_data.stack_size = 4*1024*1024;
        th_data.prio = 100;
        nErr = remote_session_control(FASTRPC_THREAD_PARAMS, (void*)&th_data, sizeof(th_data));
        if (nErr) {
                printf("ERROR 0x%x: remote_session_control failed to set thread params\n", nErr);
        } else {
                printf(" - Successfully set CDSP user process thread parameter(s)\n");
        }
        ....
        my_first_RPC_call();
}

Set stack size to 2 MB on ADSP but leave priority as default 

main() {
        struct remote_rpc_thread_params th_data;
        th_data.domain = ADSP_DOMAIN_ID;
        th_data.stack_size = 2*1024*1024;
        th_data.prio = -1;              // Pass -1 for priority to leave it as default
        nErr = remote_session_control(FASTRPC_THREAD_PARAMS, (void*)&th_data, sizeof(th_data));
        ....
        my_first_RPC_call();
}

Set priority to 120 on SLPI but leave stack size as default 

main() {
        struct remote_rpc_thread_params th_data;
        th_data.domain = SDSP_DOMAIN_ID;
        th_data.stack_size = -1;        // Pass -1 for stack size to leave it as default
        th_data.prio = 120;
        nErr = remote_session_control(FASTRPC_THREAD_PARAMS, (void*)&th_data, sizeof(th_data));
        ....
        my_first_RPC_call();
}

CPU offloading to multiple DSPs and configure thread parameters on all DSPs (priority 34 and stack size 8 MB) 

main() {
        struct remote_rpc_thread_params th_data;

        // Pass -1 for domain ID to update thread params for ALL remote subsystems the CPU process offloads to.
        th_data.domain = -1;
        th_data.stack_size = 8*1024*1024;
        th_data.prio = 0x22;
        nErr = remote_session_control(FASTRPC_THREAD_PARAMS, (void*)&th_data, sizeof(th_data));
        ....
        my_first_RPC_call();
}
