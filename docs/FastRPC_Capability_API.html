Overview

This API allows FastRPC users to query about different DSP capabilities. Since the queries need to be fulfill from the DSP side, a caching hierarchy is in place to minimize the delay per query. The first query, initiated by any user process, will request all the supported capabilities and cache all the responses. Subsequent queries will be fulfill from cache.

FastRPC DSP Capabilities

Usage

The interface for FastRPC Capability API is “remote_handle_control” and is defined on “remote.h”. 

#define DSPRPC_GET_DSP_INFO   (2)

struct remote_dsp_capability {
        uint32_t domain;       // @param[in]: DSP domain ADSP_DOMAIN_ID, SDSP_DOMAIN_ID, or CDSP_DOMAIN_ID
        uint32_t attribute_ID; // @param[in]: One of the DSP attributes from enum remote_dsp_attributes
        uint32_t capability;   // @param[out]: Result of the DSP capability query based on attribute_ID
};

int remote_handle_control(uint32_t req, void* data, uint32_t datalen);

This interface takes three arguments:

  • req[input]: uint32_t that represents the request ID. For Capability API the request ID is DSPRPC_GET_DSP_INFO.

  • data[input/output]: void* that represents a buffer given to remote_handle_control. For Capability API the data pointer is a buffer of type struct remote_dsp_capability.

  • datalen[input]: uint32_t that reprensets the size of data. For capability API it will be sizeof(struct remote_dsp_capability).

This interface returns:

  • 0 on success.

  • AEE_EUNSUPPORTEDAPI when a request ID is not supported

  • All other values as errors.

Struct remote_dsp_capability takes two input arguments and one output argument:

  • domain[input]: uint32_t that represents the DSP domain. It can be of value ADSP_DOMAIN_ID, SDSP_DOMAIN_ID, or CDSP_DOMAIN_ID.

  • attribute_ID[input]: uint32_t that represents one of the DSP attributes(capabilities) ranging from 0 to FASTRPC_MAX_DSP_ATTRIBUTES - 1.

  • capability[output]: uint32_t that will have the result of the query.

To check if your device supports this API, first declare remote_handle_control as pragma weak and then check the response and compare it to AEE_EUNSUPPORTEDAPI: 

#include "remote.h"
#pragma weak remote_handle_control      // Declare as weak symbol
...
if(remote_handle_control) {
        // Interface supported
        struct remote_dsp_capability dsp_cap = {CDSP_DOMAIN_ID, UNSIGNED_PD_SUPPORT,};
        nerr = remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
        if (nErr == (AEE_EUNSUPPORTEDAPI & 0xFF)) {
                // API not supported
        } else {
                // API supported
        }
} else {
        // Interface not supported
}

DSP Attributes

The following shows how to query each of the DSP attributes. To obtain the latest DSP attributes supported check enum table “remote_dsp_attributes” defined on remote.h from SDK.

DOMAIN_SUPPORT

  • Description: Check if DSP is supported.

  • Return value: Supported = 1, unsupported = 0.

  • Example: 

// Querying ADSP
struct remote_dsp_capability dsp_cap = {ADSP_DOMAIN_ID, DOMAIN_SUPPORT, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for DOMAIN_SUPPORT is %d", dsp_cap.capability);

UNSIGNED_PD_SUPPORT

  • Description: Check if DSP supports unsigned PD.

  • Return value: Supported = 1, unsupported = 0.

  • Example: 

// Querying CDSP
struct remote_dsp_capability dsp_cap = {CDSP_DOMAIN_ID, UNSIGNED_PD_SUPPORT, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for UNSIGNED_PD_SUPPORT is %d", dsp_cap.capability);

HVX_SUPPORT_64B

  • Description: Get how many HVX 64B are supported on the DSP.

  • Return value: Number of HVX 64B supported.

  • Example: 

// Querying CDSP for HVX 64 byte support
struct remote_dsp_capability dsp_cap = {CDSP_DOMAIN_ID, HVX_SUPPORT_64B, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for HVX_SUPPORT_64B is %d", dsp_cap.capability);

HVX_SUPPORT_128B

  • Description: Get how many HVX 128B are supported on the DSP.

  • Return value: Number of HVX 128B supported.

  • Example: 

// Querying CDSP for HVX 128 byte support
struct remote_dsp_capability dsp_cap = {CDSP_DOMAIN_ID, HVX_SUPPORT_128B, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for HVX_SUPPORT_128B is %d", dsp_cap.capability);

VTCM_PAGE

  • Description: Get the maximum possible page size (in bytes) allocation in VTCM.

  • Return value: Maximum possible page size.

  • Example: 

// Querying ADSP for VTCM page size
struct remote_dsp_capability dsp_cap = {ADSP_DOMAIN_ID, VTCM_PAGE, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for VTCM_PAGE is %d", dsp_cap.capability);

VTCM_COUNT

  • Description: Get the number of page_size blocks in VTCM.

  • Return value: Number of page_size blocks in VTCM.

  • Example: 

// Querying ADSP for VTCM page count
struct remote_dsp_capability dsp_cap = {ADSP_DOMAIN_ID, VTCM_COUNT, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for VTCM_COUNT is %d", dsp_cap.capability);

ARCH_VER

  • Description: Get Hexagon processor architecture version.

  • Return value: DSP architecture number.

  • Example: 

// Querying CDSP
struct remote_dsp_capability dsp_cap = {CDSP_DOMAIN_ID, ARCH_VER, 0};
remote_handle_control(DSPRPC_GET_DSP_INFO, &dsp_cap, sizeof(struct remote_dsp_capability);
printf("Result of capability query for ARCH_VER is %d", dsp_cap.capability);
