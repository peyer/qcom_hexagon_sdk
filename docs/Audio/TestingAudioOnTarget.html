Overview

A dynamic library generated and tested on simulator earlier can be loaded on target (Android device).

The steps are summarized below

  1. Update the ACDB (Audio Calibration Database) with details of custom topology containing new module

  2. Pushing the module to Target

  3. Conduct real time calibration of the audio module

Get started on QACT Lite

QACT (Qualcomm Audio Calibration Tool) Lite helps you load the custom module on target by updating the ACDB. See the page Target Testing : QACT to get started.

Pushing the module to Target

The dynamic object is pushed to target. See the section Audio Module Addition

Real Time Calibration of Module

There are two methods for real time calibration of module.

  1. Using QACT Lite.

Using QACT Lite

QACT Lite provides a GUI based real time calibration of audio modules. Refer QACT Lite page for details.

Using audiocal Java application

NOTE : Real time calibration using java application is supported for APQ8094 dragonboard only.

Real time calibration using java application is supported now. More detailed information is available at audio calibration using app page

This section explains about the usage of android java application (audiocal.apk) which is a part of SDK and can set/get params of a module.

Following steps will perform real time calibration of module parameters:

  1. Push ADSP split binaries to /etc/firmware and reboot the device.

  2. adb root

  3. adb install <HEXAGON_SDK_ROOT>/lib/audio/audiocal/audiocal.apk

  4. Add topology, module and parameter information to <HEXAGON_SDK_ROOT>/lib/audio/audiocal/audiocal.xml 

    <!-- Modules data -->
<module name="fir_filter" id="0x00000001">
    <param name="fir_enable" id="0x12D03">
        <value>0x1</value>
    </param>        
    <param name="fir_disable" id="0x12D03">
        <value>0x0</value>
    </param>        
    <param name="fir_highpass_coeff" id="0x12D05">
        <value>0x0000000D,0xF838F9A3,0xF5D2F6EC,0xF474F4FB,0xF4747501,0xF5D2F4FB,0xF838F6EC,0xFB1AF9A3</value>
    </param>        
    <param name="fir_lowpass_coeff" id="0x12D05">
        <value>0x0000000D,0x001DFFF3,0x016BFB99,0x0C08FA71,0x0C08309A,0x016BFA71,0x001DFB99,0x0000FFF3</value>
    </param>        
</module>


<!-- Topologies data -->
<topology name="custom_topology" id="0x100000FD">
    <module name="fir_filter" id="0x00000001" />
</topology>

  5. adb remount

  6. adb push <HEXAGON_SDK_ROOT>/lib/audio/audiocal/audiocal.xml /data/

  7. Start audio playback (preferably loop it)

  8. Launch “Audiocal” application in device

  9. Press “LOAD AUDIO CAL DATABASE” button

  10. To set/get params :

    1. Select Device: AUDIO_DEVICE_OUT_SPEAKER

    2. Select AppType: AUDIO_PLAYBACK

    3. Select calType: AUDIO_COPP

    4. Select Sampling: 48000 (might change based on your topology and device)

    5. Select Topology: custom_topology

    6. Select Module: fir_filter

    7. Select param: fir_coeff

    8. Don't set Persistent for real time calibration

    9. Press “Set”

    10. You can verify the data set or not by pressing “Get” after “Set”

    11. You can see the effect of setting low pass and high filter coeff
